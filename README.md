# Project #032: Fama-French 3-Factor Decomposition Engine

An institutional-grade risk analytics engine designed to decompose quantitative trading strategy returns into systemic risk premiums using multi-variable Ordinary Least Squares (OLS) linear regression. 

This engine acts as a performance gatekeeper, mathematically proving whether a strategy's returns are driven by a genuine idiosyncratic edge ($\alpha$) or if they are simply a passive byproduct of accidental factor exposures ($\beta$).

---

## 🧠 Core Mathematical Theorem

Traditional Capital Asset Pricing Model (CAPM) architectures assume portfolio returns are driven solely by baseline market sensitivity ($\beta$). The Fama-French Three-Factor Model (1993) expands this by introducing capitalization size and value matrices to explain over 90% of diversified portfolio variance.

The engine fits the following multi-variable linear regression equation:

$$R_p(t) - R_f(t) = \alpha + \beta_{Mkt}[R_m(t) - R_f(t)] + \beta_{SMB}SMB(t) + \beta_{HML}HML(t) + \epsilon(t)$$

### Vector & Variable Definitions:
* **$R_p(t)$**: The daily return vector of your trading strategy at time $t$.
* **$R_f(t)$**: The daily Risk-Free Rate of return (sourced from US Treasury Bill rates).
* **$R_p(t) - R_f(t)$**: The **Dependent Variable ($Y$)**, representing the strategy's excess returns.
* **$\alpha$ (Alpha)**: The Y-intercept constant. This represents the premium generated by your strategy that *cannot* be explained by passive factor loading.
* **$\epsilon(t)$**: The residual error term at time $t$.

---

## 📊 The Factor Risk Premiums

The independent variable matrix ($X$) is built from three distinct macro risk premiums computed daily by the Kenneth French Data Library:

1.  **Market Risk Premium ($Mkt - RF$)**: The excess return of the capitalization-weighted market portfolio over the risk-free rate.
2.  **Size Premium / SMB (Small Minus Big)**: The historical asset anomaly where small-cap stocks outperform large-cap stocks. It is calculated by taking the average return of three small-cap portfolios and subtracting the average return of three large-cap portfolios.
3.  **Value Premium / HML (High Minus Low)**: The structural anomaly where value stocks (high book-to-market ratios) outperform growth stocks (low book-to-market ratios).

## 🧮 Statistical Validation & Interpretation Metrics

When evaluating the OLS summary printout, the engine uses strict statistical boundaries to prevent confirmation bias:

### 1. Alpha Significance via $p$-values
* **The Problem:** A strategy can print a positive annualized alpha purely due to lucky sample variance.
* **The Math:** We test the Null Hypothesis ($H_0: \alpha = 0$). The engine calculates a $t$-statistic for the intercept. 
* **The Threshold:** We enforce a strict significance level of **$p \le 0.05$**. If the $p$-value of your Alpha is greater than $0.05$, the strategy's edge is mathematically unproven, meaning the performance is highly likely to be a statistical fluke.

### 2. Adjusted R-Squared ($R^2_{adj}$)
Unlike standard R-squared, the Adjusted R-squared penalizes the inclusion of unnecessary explanatory variables:

$$R^2_{adj} = 1 - \left[ \frac{(1 - R^2)(n - 1)}{n - k - 1} \right]$$

Where $n$ is the number of trading days (data samples) and $k$ is the number of factors ($k=3$). A high $R^2_{adj}$ (e.g., $>0.80$) means your strategy's returns are heavily locked to systematic factor currents.

### 3. Data Scaling Guardrail
Kenneth French publishes data using raw integer percentages (e.g., a return of $1.5\%$ is written as `1.50`). Standard strategy logs record daily returns as decimals (e.g., $1.5\%$ is written as `0.015`). 
* **CRITICAL FIX:** This engine contains an explicit scaling layer that divides French's data vectors by $100$ before matrix multiplication to prevent factor beta calculations from scaling incorrectly by a factor of 100.

## 🛠️ System Architecture Directory Map

The system is fully decoupled to ensure mathematical components can be plugged into any asset backtesting ledger:
* `data_fetcher.py`: Automates raw web scraping, ZIP decompression, string buffer handling, and datetime alignment against Dartmouth's structural data files.
* `factor_engine.py`: Isolates overlapping indices, constructs the $X$ and $Y$ matrices, introduces the OLS intercept constant, and fires the multi-variable regression.
* `run_analysis.py`: The master execution script that runs validation loops and outputs high-density quantitative briefs.
